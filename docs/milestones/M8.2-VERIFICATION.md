# M8.2: Verification System

## Goal

Mechanically enforce code quality checks after Claude completes tasks, using Claude Code hooks to block task completion when checks fail. Replaces instruction-based verification (telling Claude to verify) with hook-based enforcement (exit code 2 blocks the task).

## Depends On

M1 (Hook Protocol), M4 (Server & Dashboard — for SSE broadcasting of verification events)

---

## Completed

### What Changed

**Added mechanical verification via TaskCreated/TaskCompleted/Stop hooks.** The system snapshots file state when a task begins, diffs against the snapshot when the task completes, and runs only the checks whose globs match changed files. Failed checks exit with code 2, which Claude Code interprets as a blocking failure — Claude must fix the issues before the task can complete.

### Architecture

```
TaskCreated hook                    TaskCompleted / Stop hook
      │                                      │
      ▼                                      ▼
restruct snapshot                    restruct verify
      │                                      │
      ▼                                      ▼
Load .restruct/verify.yaml          Load snapshot from SQLite
      │                                      │
      ▼                                      ▼
git ls-files + glob filter           Diff mtime/size vs current
      │                                      │
      ▼                                      ▼
Record mtime + size → SQLite         Filter checks by changed files
                                             │
                                             ▼
                                     Run matching checks (fail-fast)
                                             │
                                     ┌───────┴───────┐
                                     │               │
                                  Pass            Fail
                                     │               │
                                  exit 0          exit 2
                                  promote       Claude must fix
                                  snapshot       before completing
```

### Verification Config

Per-project config at `.restruct/verify.yaml`:

```yaml
checks:
  - name: test
    command: "pnpm test"
  - name: typecheck
    command: "pnpm --filter web exec tsc --noEmit"
    globs:
      - "web/**/*.ts"
      - "web/**/*.tsx"
  - name: go-vet
    command: "pnpm --filter @restruct/cli exec go vet ./..."
    globs:
      - "cli/**/*.go"
```

- Checks with globs only run when changed files match those patterns
- Checks without globs always run when any file changes
- Checks execute sequentially, fail-fast on first failure
- Each check has a 110s timeout

### Hook Wiring (plugin.json)

| Hook Event | Command | Timeout | Purpose |
|---|---|---|---|
| TaskCreated | `restruct snapshot` | 10s | Baseline file state |
| TaskCompleted | `restruct verify` | 120s | Run checks, block on failure |
| Stop | `restruct verify` | 120s | Run checks on user stop |

### Snapshot System

- File discovery via `git ls-files --cached --others --exclude-standard` (respects .gitignore)
- Falls back to directory walk if git unavailable
- Records relative path, mtime (UnixNano), and file size per file
- Scope system: "prompt" for UserPromptSubmit baseline, task ID for TaskCreated
- On verification pass, task scope is promoted to "prompt" scope
- Stale snapshots (>24h) are pruned automatically

### Diff Detection

Compares snapshot entries against current file stat:
- **Modified:** mtime or size changed
- **Deleted:** file in snapshot no longer exists
- **New:** file in snapshotted directory not in snapshot

### Loop Prevention

The `Stop` hook also runs verification, but checks `input.StopHookActive` — if true, verification is skipped to prevent infinite loops when Claude tries to stop despite check failures.

### Database Schema

Two tables added via migrations 005-007:

**snapshots:**
| Column | Type | Purpose |
|--------|------|---------|
| session_id | TEXT | Claude Code session |
| scope | TEXT | "prompt" or task ID |
| file_path | TEXT | Relative path |
| mod_time | INTEGER | Unix nanoseconds |
| file_size | INTEGER | Bytes |

**verification_events:**
| Column | Type | Purpose |
|--------|------|---------|
| session_id | TEXT | Claude Code session |
| refinement_id | INTEGER (nullable) | Links to refinement |
| scope | TEXT | "prompt" or task ID |
| hook_event | TEXT | Which hook triggered this |
| event_type | TEXT | "snapshot" or "verify" |
| file_count | INTEGER | Files in snapshot |
| changed_files | TEXT (JSON) | List of changed paths |
| checks_run | TEXT (JSON) | Check results |
| result | TEXT | "pass", "fail", or "skip" |

### SSE Broadcasting

Verification events are broadcast to the dashboard in real-time:
- `verification:snapshot` — file snapshot recorded
- `verification:check-result` — check pass/fail with details

Events are POSTed to `http://localhost:8377/api/verification` and relayed via SSE hub to all connected dashboard clients. Non-blocking — HTTP happens in a background goroutine.

### Web Dashboard Integration

The RefinementDetail page shows verification events in a dedicated tab alongside pipeline events. Each event shows scope, result, changed file count, and individual check results.

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Enforcement mechanism | Exit code 2 (hook blocking) | Mechanical enforcement > instruction compliance. Claude cannot ignore a blocked hook |
| Snapshot storage | SQLite (shared DB) | Consistent with M4 data layer. Enables dashboard queries |
| File discovery | `git ls-files` with fallback | Respects .gitignore, captures tracked + untracked files |
| Check execution | Sequential, fail-fast | First failure is the priority. Parallel would complicate output |
| Scope promotion | Task → prompt on pass | Successful task snapshot becomes the new baseline |
| Glob matching | Per-check globs filter which checks run | TypeScript changes shouldn't trigger Go vet |
| Config location | `.restruct/verify.yaml` | Per-project, gitignored by default, alongside other restruct state |
| asyncRewake scope | TaskCompleted only | Stop must be synchronous (block until pass). SubagentStop (M16) must be synchronous (block before returning to parent). TaskCompleted is the only event where async makes sense — Claude can start the next task while checks run |
