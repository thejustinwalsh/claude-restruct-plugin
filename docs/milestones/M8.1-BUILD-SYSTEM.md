# M8.1: Build System Migration (Make → xmake + pnpm)

## Goal

Replace the ad-hoc Makefiles with a proper build system that has dependency tracking, incremental builds, and cross-compilation support. Provide a single-command dev workflow and a single-command release pipeline.

## Depends On

None — infrastructure change, no feature dependencies.

---

## Completed

### What Changed

**Replaced Make with xmake + pnpm workspaces.** The project now uses xmake (installed via npm `xmake-build-system`) as the build orchestrator, with pnpm workspaces providing a unified script interface across the Go CLI and React web dashboard.

### Build Graph

```
web/src/**  ───────────────────→  [vite build]  ──→  web/dist/
                                                        │
                                                    [cp -r]
                                                        │
                                                        ▼
cli/**/*.go                                    cli/internal/server/dist/
cli/go.mod                                         (go:embed)
cli/internal/prompt/*.tmpl         ──────────→  [go build]  ──→  plugin/bin/restruct
cli/internal/db/migrations/*.sql
```

All targets use `depend.on_changed()` for incremental builds — unchanged inputs skip the build step entirely (~0.01s no-op).

### Debug vs Release

Go build tags control runtime behavior:
- **Debug** (`-tags debug`): `WebDist()` returns nil, server enables CORS for Vite dev proxy
- **Release** (default): `WebDist()` returns embedded `go:embed` filesystem

Configured via `xmake f -m debug` / `xmake f -m release` (persists until reconfigured).

### Commands

| Command | What it does |
|---------|-------------|
| `pnpm dev` | Sets debug mode, starts Go server (with file watch + auto-restart) and Vite HMR in parallel |
| `pnpm build` | Sets release mode, builds all 3 platform binaries (darwin-arm64, darwin-x86_64, linux-x86_64) |
| `pnpm test` | Runs test scripts across all workspaces |
| `pnpm clean` | Removes all build artifacts |

### Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Build orchestrator | xmake (via npm) | Lua-scriptable Make replacement with dependency tracking. Simple, no cache bloat (vs Turbo), no complex config (vs Nx) |
| Package manager | pnpm workspaces | Unified script interface, workspace members for both web/ and cli/ |
| Go build approach | Phony targets with `go build` | xmake's native Go support (`set_languages("go")`) is broken for Go 1.20+ ([xmake-io/xmake#3586](https://github.com/xmake-io/xmake/issues/3586)). Corrupts GOROOT via `make.bash` bootstrap |
| Binary target metadata | `set_kind("binary")` without `add_files` | Gives xmake the output path (enables `xmake run`) without triggering Go toolchain bootstrap |
| Debug/release mode | Go build tags (`//go:build debug`) | Compile-time switch, no runtime flag needed. One binary, correct behavior |
| Cross-compilation | `GOOS`/`GOARCH` env vars in phony targets | Go handles cross-compilation natively, no toolchain bootstrap needed |
| Dev workflow | `xmake watch -d cli -r` + Vite HMR | File watch triggers rebuild + restart for Go, Vite handles web HMR. Single `pnpm dev` command |
| Custom DSL | `xmake/phony.lua` + `xmake/go_build.lua` | `import("mod", {inherit = true})` injects `sources()`, `on_changed()`, `go()` into callbacks. Reads like built-in xmake |

### Known Limitation

xmake's `set_languages("go")` and native Go toolchain support is broken for Go 1.20+ installed via package managers (Homebrew, apt). It attempts to bootstrap the Go compiler via `make.bash`, which deletes tool binaries from GOROOT and fails on binary distributions. **Do not use `set_languages("go")` or `add_files("*.go")` with Go targets.** Use phony targets with direct `go build` calls instead.

## Release Targets

| Target | Group | Platform | Output |
|--------|-------|----------|--------|
| `cli` | default | host | `plugin/bin/restruct` |
| `cli-darwin-arm64` | release | macOS ARM | `plugin/bin/restruct-darwin-arm64` |
| `cli-darwin-x86_64` | release | macOS Intel | `plugin/bin/restruct-darwin-x86_64` |
| `cli-linux-x86_64` | release | Linux x64 | `plugin/bin/restruct-linux-x86_64` |

`pnpm build` runs `xmake build -g release` which builds all release group targets. The DAG ensures web → copy-web-dist → Go cross-compile runs in order, skipping unchanged steps.
